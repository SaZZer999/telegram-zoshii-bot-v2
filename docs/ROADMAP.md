# ROADMAP.md

## Поточний стан: готова основа

Household Core + Aliases + Expenses v1 + Global Household Router вже працюють і реально використовуються:

* спільні покупки й спільні запаси;
* структуровані кількості (structured quantities);
* preview перед кожною важливою зміною;
* природні команди з відкритого списку;
* часткове списання запасів;
* звіряння (reconciliation) повного списку запасів;
* persistent context відкритого списку після restart/deploy;
* домашні назви товарів (aliases v1), з пріоритетом над вбудованими синонімами;
* витрати (expenses v1): додавання, останні витрати, місячний підсумок, видалення однієї витрати;
* Global Household Router v1: змішані команди («Купив X за Y zł» тощо) одним спільним preview і однією транзакцією, незалежно від відкритого меню.

Як саме це працює й на чому базується — детально в `docs/PROJECT.md`. Це робоча основа, а не остаточно завершений проєкт: нижче йде послідовне розширення функціоналу.

## Правило роботи над roadmap

Один етап за раз. Перед новим великим етапом:

1. переконатися, що попередній етап реально працює;
2. зафіксувати поточний стан комітом;
3. коротко погодити план змін;
4. не змішувати кілька великих функцій в одному завданні.

## Етап 1 — Реальне користування та стабілізація

* Користуватися ботом у звичайному житті, включно з витратами й Global Household Router.
* Додавати в роботу лише підтверджені реальні баги або незручності.
* Не запускати великі рефакторинги без фактичної причини.
* Не додавати функції "про всяк випадок".

## Етап 2 — Контрольований refactor `bot.py`

* `bot.py` уже частково розвантажено (`expenses.py`, `household_router.py`); далі — обережно, невеликими кроками, без зміни поведінки.
* Виносити логіку в окремі модулі лише по одній чіткій межі за раз (наприклад покупки або запаси окремо).
* Кожен крок — з робочими тестами до і після, без зміни зовнішньої поведінки бота.

## Етап 3 — Розширення витрат

* Редагування вже записаної витрати.
* Довільні звітні періоди (не лише поточний місяць).
* Простий експорт витрат.

## Етап 4 — Рецепти та меню

* Ідеї страв з актуальних запасів.
* Продукти, які варто використати першими.
* Створення покупок із рецепта через preview.
* Меню на кілька днів.

## Етап 5 — Терміни придатності

* Дата купівлі.
* Дата відкриття.
* Нагадування.
* Список продуктів, які варто використати першими.

## Етап 6 — Необов'язкові розширення

* Persistent pending previews.
* Голосові повідомлення.
* Фото чеків і OCR.
* Фото холодильника.
* Вебпанель — лише тоді, коли Telegram стане недостатнім.

## Визначення готовності

* **Household Core готовий** — покупки й запаси працюють безпечно у щоденному використанні.
* **Household Assistant v1 готовий** — Household Core + Aliases + Expenses v1 + Global Household Router (поточний стан).

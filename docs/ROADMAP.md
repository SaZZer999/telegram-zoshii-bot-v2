# ROADMAP.md

## Правило роботи

Кожен етап виконується окремо.

Перед початком нового етапу потрібно:

1. Перевірити, що попередній етап працює.
2. Зробити локальний Git commit.
3. Погодити короткий план змін.
4. Не змішувати кілька великих функцій в одному завданні.

---

## Stage 0 — Стабілізація та безпека

### Мета

Зрозуміти реальний стан коду та зробити базові безпечні налаштування.

### Завдання

* [x] Провести read-only аудит репозиторію.
* [x] Додати `.env`, `venv/` і `*.db` у `.gitignore`.
* [x] Створити `CLAUDE.md`, `docs/PROJECT.md` і `docs/ROADMAP.md`.
* [ ] Перевірити Render Dashboard:

  * Build Command;
  * Start Command;
  * Auto-Deploy;
  * назви Environment Variables без показу значень.
* [ ] Перевірити, чи старий Telegram token не був показаний у повідомленнях або логах.
* [ ] Перед наступним deploy оновити Telegram token і замінити token у webhook URL на окремий webhook secret.
* [ ] Створити локальний Git commit для документації.

### Результат

Є зрозумілий опис проєкту, базові правила безпеки та перевірена схема деплою.

## Визначення готового MVP

MVP готовий, коли:

1. Бот доступний лише двом користувачам.
2. Є робоче Telegram-меню.
3. Дані не зникають після deploy або restart.
4. Є спільний список покупок.
5. Є прості запаси продуктів.
6. Є запис покупок і витрат.
7. Бот пропонує страви з наявних продуктів.
8. Кожну важливу зміну даних користувач підтверджує вручну.

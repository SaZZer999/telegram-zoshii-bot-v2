# PROJECT.md

## 1. Що це за проєкт

Це приватний Telegram-помічник для одного домашнього господарства: Максима та його дружини.

Його завдання — допомагати з повсякденними домашніми справами: покупками, приблизним обліком продуктів, витратами та ідеями страв.

Проєкт робиться для особистого використання. Це не публічний сервіс, не стартап і не продукт для інших користувачів.

Єдиний інтерфейс першої версії — Telegram.

## 2. Поточний стан

Наразі бот уже працює на Render як Python Flask Web Service.

Поточна логіка:

1. Telegram надсилає webhook-запит до Flask.
2. `bot.py` приймає текстове повідомлення.
3. Текст надсилається до Groq.
4. Бот повертає відповідь у Telegram.

Відомі обмеження:

* Немає Telegram-кнопок або `/start`.
* Немає постійної бази даних.
* Історія чатів зберігається тільки в RAM.
* Після restart, redeploy або засинання Render історія губиться.
* `database.py` поки порожній.
* Бот поки не має списку покупок, запасів або витрат.

## 3. Мета MVP

Швидко створити корисного Telegram-помічника для двох людей.

Після завершення MVP бот повинен уміти:

1. Вести спільний список покупок.
2. Вести простий приблизний список продуктів удома.
3. Додавати покупки та записувати витрати у zł.
4. Показувати, що є вдома.
5. Пропонувати, що можна приготувати з наявних продуктів.
6. Працювати лише для Максима та його дружини.
7. Використовувати українську мову інтерфейсу.

## 4. Основні сценарії

### Список покупок

Користувач пише:

* "Додай молоко до покупок".
* "Додай 2 упаковки яєць".
* "Покажи покупки".
* "Купив молоко".

Бот повинен дозволяти:

* додати позицію;
* показати список;
* позначити позицію як куплену;
* видалити позицію;
* зберегти кількість та одиницю, якщо вони відомі.

### Запаси вдома

Користувач може:

* додати продукт у запаси;
* переглянути запаси;
* вказати приблизну кількість;
* відредагувати або видалити продукт.

Приклади:

* "У нас є 2 літри молока".
* "Додай 500 г курки в запаси".
* "Покажи, що є вдома".

### Покупка та витрата

Приклад повідомлення:

"Купив 2 літри молока за 8,99 zł".

Бот повинен:

1. Показати, як він зрозумів повідомлення.
2. Попросити підтвердження.
3. Після підтвердження:

   * додати товар у запаси;
   * позначити пункт у списку покупок як куплений, якщо він там є;
   * записати витрату.

### Ідеї страв

Користувач може написати:

* "Що можна приготувати?"
* "Запропонуй швидку вечерю".
* "Що використати зі шпинатом?"

Бот передає список поточних запасів у Groq і отримує короткі пропозиції страв.

На першому етапі бот не списує продукти автоматично після пропозиції рецепта.

## 5. Обмеження першої версії

У v0.1 свідомо не реалізовуємо:

* вебпанель;
* окремий мобільний застосунок;
* публічну реєстрацію;
* кілька домогосподарств;
* складну бухгалтерію;
* банківські інтеграції;
* автоматичне розпізнавання холодильника з фото;
* фото чеків;
* голосові повідомлення;
* календар;
* розумний дім;
* аптечку;
* автоматичне списання продуктів без підтвердження.

## 6. Технічні принципи

* Telegram залишається єдиним інтерфейсом.
* Поточна Flask webhook-архітектура зберігається.
* Дані мають зберігатися у хмарній PostgreSQL-базі після її підключення.
* Зміни покупок, запасів і витрат мають підтверджуватися користувачем.
* Дані доступні лише двом дозволеним Telegram-користувачам.
* Усі суми зберігаються в польських злотих.
* Кількості продуктів можуть бути приблизними.
* Бот не повинен створювати ілюзію точного харчового чи фінансового обліку.

## 7. Майбутні функції після MVP

Лише після завершення MVP можна розглядати:

* дати купівлі;
* дати відкриття;
* нагадування про термін придатності;
* фото чеків;
* голосове введення;
* меню на кілька днів;
* рецепти зі списком інгредієнтів;
* списання продуктів після підтвердження "приготував".
